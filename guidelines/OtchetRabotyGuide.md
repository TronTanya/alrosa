# Гайд: отчёт о проделанной работе (со скриншотами)

Документ помогает собрать **наглядный отчёт** (Word, PDF, презентация): что снять, в каком порядке, как назвать файлы. Скрины храните в папке `guidelines/report-screenshots/` (PNG или WebP, без лишнего сжатия).

---

## 1. Как делать скриншоты (Windows)

| Действие | Как |
|----------|-----|
| Область экрана | `Win + Shift + S` → выделить область → вставить в документ или сохранить из буфера |
| Окно целиком | `Alt + Print Screen` (активное окно) |
| Полный экран | `Win + Print Screen` (сохраняется в «Изображения\Снимки экрана») |
| Длинная страница | Расширение браузера «Full Page Screen Capture» или DevTools → `Ctrl+Shift+P` → «Capture full size screenshot» (Chrome) |

**Рекомендации:** одинаковый масштаб браузера (например 100%), светлая тема, без личных данных в кадре; при необходимости замажьте ФИО в графическом редакторе.

---

## 2. Именование файлов

Шаблон: `NN-роль-кратко.png`, например:

- `01-employee-home.png`
- `05-hr-dashboard.png`
- `12-admin-monitoring.png`

Нумерация по порядку вставки в отчёт.

## 4. Структура текстовой части отчёта

1. **Цель и период** — что делали и за какой срок.
2. **Среда** — фронт: Vite + React; при необходимости ссылка на `README` / версию Node.
3. **Результат по блокам** — для каждого роли или фичи: 1–2 абзаца + скрин из списка выше.
4. **Риски и ограничения** — известные баги, заглушки, mock-данные.
5. **Следующие шаги** — что планируется в следующей итерации.

## 6. Краткий аудит «чистоты» кода (на дату проверки)

Проверено автоматической сборкой и линтером IDE.

| Критерий | Статус |
|----------|--------|
| Сборка `npm run build` | Успешно |
| Ошибки в IDE (TypeScript) по `src/app` | Не обнаружены |
| `console.log` / `console.debug` в `src` | Не найдены (хорошо для продакшена) |
| Использование `any` | В кастомных тултипах и активном секторе Pie — заменено на `TooltipProps` из `recharts` и явный тип сектора (`LearningAnalyticsChart`, `SystemOverviewChart`) |
| Размер бандла | Предупреждение Vite: один чанк > 500 KB — при необходимости дальше: `import()` для тяжёлых страниц |
